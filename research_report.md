# Technical Research Report: R-Rated Content Filtering Stack

## Overview
This report outlines the recommended open-source technical stack for building software that can automatically detect and filter NSFW content (nudity, violence, and profanity) in movies to make them "safe for kids."

## 1. Visual Content Filtering (Nudity & NSFW)
The primary challenge is detecting nudity and sexual content frame-by-frame.

### Recommended Models:
*   **NudeNet (bedapudi6788/NudeNet)**:
    *   **Type**: Object Detection & Classification.
    *   **Pros**: Can detect specific body parts (exposed breasts, genitals, etc.). This allows for "smart blurring" where only the offending area is obscured rather than the entire frame.
    *   **Usage**: Python-based, supports ONNX for performance.
*   **viddexa/nsfw-detection-2-mini (Hugging Face)**:
    *   **Type**: Image Classification.
    *   **Pros**: Very fast and modern. Good for a first-pass check to see if a scene contains NSFW content.
*   **Yahoo OpenNSFW**:
    *   **Type**: Classification.
    *   **Pros**: Lightweight, classic baseline.

## 2. Violence Detection
Violence detection is temporally sensitive (a single frame might not look violent, but a sequence of frames does).

### Recommended Models:
*   **YOLOv8 (fine-tuned)**:
    *   **Pros**: Extremely fast (real-time). Can be trained to detect weapons (knives, guns) and violent actions.
*   **CNN + LSTM (e.g., hasnainnaeem/Violence-Detection-in-Videos)**:
    *   **Pros**: Analyzes sequences of frames to identify fights, explosions, or aggressive movements.
    *   **Dataset Reference**: RWF-2000 is a standard dataset for training these models.

## 3. Audio Content Filtering (Profanity)
Filtering audio requires converting speech to text and then matching against a profanity database.

### Recommended Stack:
*   **Speech-to-Text**: **OpenAI Whisper (Open-Source)**. It is highly accurate and provides timestamps for every word, which is critical for precise muting or "beeping."
*   **Profanity Identification**: **profanity-check** or **better-profanity** (Python).
*   **Audio Manipulation**: **FFmpeg** or **MoviePy** to mute the specific time ranges identified by Whisper.

## 4. Video Processing & Orchestration
The "glue" that brings it all together.

### Libraries:
*   **FFmpeg**: The essential tool for demuxing, muxing, and applying filters at scale.
*   **MoviePy**: A Python library that makes it easy to cut, concatenate, and apply effects (like blur) to video clips. It is ideal for "skipping" or "muting" segments by concatenating the "safe" parts of the video.
*   **OpenCV**: Useful for frame-by-frame manipulation and applying custom masks generated by detection models.

## Recommended Tech Stack (Summary)
*   **Language**: Python 3.10+
*   **Video Processing**: MoviePy + FFmpeg
*   **Visual NSFW**: NudeNet (Object Detection)
*   **Violence**: YOLOv8 (Action Detection)
*   **Audio STT**: Whisper (Medium or Large model)
*   **Filtering Logic**: Custom Python script coordinating frame extraction -> model inference -> frame blurring/audio muting or skipping -> re-encoding.

## Proposed Workflow
1.  **Extract Audio**: Use FFmpeg to pull audio from the video.
2.  **Transcribe Audio**: Run Whisper on the audio to get a timestamped transcript.
3.  **Identify Profanity**: Flag timestamps in the transcript that contain banned words.
4.  **Scene/Frame Analysis**:
    *   Sample frames from the video (e.g., 2-5 fps).
    *   Run NudeNet and Violence models on sampled frames.
    *   If a frame is flagged, analyze surrounding frames to determine the start/end of the scene.
5.  **Edit Video**:
    *   **Muting**: Use MoviePy to zero the volume of flagged audio segments.
    *   **Blurring**: Use OpenCV/MoviePy to apply blur masks to specific regions.
    *   **Skipping**: Use MoviePy to cut the flagged scenes entirely and concatenate the remaining segments.
6.  **Re-encode**: Export the cleaned video.
